Re-rendering

  • to see re-renders visually,
    inspect ⇒ components (by react developer tools) ⇒ settings icon ⇒ check the ‘
    Highlight updates when the components render’
  • change inspect window dock side - 3 dots on top right corner
  • React.memo() - used to define a child component which will not re-render upon re-rendering of its parent component if the props are unchanged
    // ----Parent Component----
const Counter = () => {
  const [count, setCount] = useState(0);

  const handleClick = () => {
    setCount(count + 1);
  }

  return (
    <div>
			<Header title="Counter App" />
      <button>Count: {count}</button> 
    </div>
  );
};

// ----Child Component----
// normal child component
const Header = (title) => {
	return <h1>{title}</h1>;
}

// child component using memo()
const Header = React.memo((title) => {
	return <h1>{title}</h1>;
});

Keys

  • key helps react to identify a dom element from an array of elements, which helps re-render all those elements more optimally.
    • react will show the desired output without keys as well
    • without keys react will remove the old array and insert a new one
    • with keys react can tell which element(s) from array need to change or move and make only required changes

HOOKS

  • component’s lifecycle -
    • mounting - component rendered on the screen first time
    • upadating - re-rendering
    • unmounting - component is being removed from the DOM
  • class based components used to use state & lifecycle methods to perform certain actions or side effects at specific points of component’s lifecycle.
    • side effects - Anything that is not related to rendering, putting things on the DOM, taking things out of the DOM are called side effects.
      Things like you are trying to interact with the browser, fetching data from the backend.
  • hooks provide a way to manage state, perform side effects and use other react features within functional components.
    • useState and useContext - to manage state
    • useEffect - to perform side effects
    • useCallback, useMemo, useRef, etc gives functional components additional features
  • lifecylce methods and hooks

useEffect

  • use useEffect when fetching data from a back-end
    function Todos() {
	const [todos, setTodos] = useState([]);

	useEffect(() => {
		axios
			.get("backend.url")
			.then((res) => setTodos(res.data.todos));
	}, []);
}
  • If this request to back-end is not made inside the useEffect, what will happen is-
    a request goes to back-end, a response comes, we update todos with the response, state change causes re-render, function Todos start to execute again line by line, it again encounters the request to the back-end and it forms an infinite-loop

Some More hooks

  • useMemo - used to remember value of an expensive operation so that react does not perform the same expensive operation on every re-render
    function Component({a, b}) {	
	const value = a + b;  // ---------------------------(1)

	const value = useMemo(() => { //---------------------------(2)
		return a + b;
	},[a, b]);
	
	return <div>{value}</div>;
}

    in case 1, value will be computed everytime component re-renders
    in case 2,
    value will be computed only when a || b changes

  • useCallback - used in conjuction with React.memo() when passing a function as a prop
    let a = 5;
let b = 5;
a === b // true

let str1 = "Ankit";
let str2 = "Ankit";
str1 === str2 // true

let arr1 = [1, 2];
let arr2 = [1, 2];
arr1 === arr1 // false

let obj1 = {};
let obj2 = {};
obj1 === obj2 // false

function sum1(a, b) {return a + b;}
function sum2(a, b) {return a + b;}
sum1 === sum2 // false

    In case of arrays, objects and functions, JS does not compare their value but their reference.
    sum1 and sum2, arr1 and arr2, obj1 and obj2 —— they are not same referentially, i.e. they are pointing to different blocks in memory.

    • React.memo() - used to define a child component which will not re-render upon re-rendering of its parent component if the props are unchanged
    • But if functions are passed as props, React.memo() alone doesn’t work.
      const ParentComponent = () => {
  const [count, setCount] = useState(0);

  const handleClick = () => {
    setCount(count + 1);
  };

  const sum = (a, b) => {
    return a + b;
  };

  return (
    <div>
      <button onClick={handleClick}>Count: {count}</button>
      <ChildComponent sum={sum} />
    </div>
  );
};

const ChildComponent = React.memo(({ onClick }) => {
  console.log('Child component rendered');
  return <button onClick={onClick}>Click me</button>;
});

      Because eveytime ParentComponent re-renders, a new sum function is created with same defination but different reference.

    • We can use useCallback hook to optimize performance in such cases
      const ParentComponent = () => {
  const [count, setCount] = useState(0);

  const handleClick = () => {
    setCount(count + 1);
  };

  const sum = useCallback((a, b) => {
    return a + b;
  }, []);

  return (
    <div>
      <button onClick={handleClick}>Count: {count}</button>
      <ChildComponent sum={sum} />
    </div>
  );
};

const ChildComponent = React.memo(({ onClick }) => {
  console.log('Child component rendered');
  return <button onClick={onClick}>Click me</button>;
});
    • The sum function is memoized, and it will only be recreated if any of the dependencies in the array change.

Custom hooks

  • Used to hide complex state management, side effects, etc from the components, so that they are primarily focused on returning markups and user interaction
    function App() {
	const todos = useTodos(); // returns an array of todos

	return (
		<>
			{todos.map((todo) => (
	        <Todo key={todo.id} title={todo.title} description={todo.description} />
	      ))}
		</>
	);
}
    
const useTodos = () => {
	const [todos, setTodos] = useState([])

	useEffect(() => {
	  axios.get("https://sum-server.100xdevs.com/todos")
	    .then((res) => setTodos(res.data.todos))
	}, []);
	
	return todos;
}
  • We can use this extracted logic across different components and it Improves readability

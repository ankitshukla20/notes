className in JSX is the replacement for class in HTML.

class is a reserved keyword in javascript.

CSS stylesheets
  • create a CSS file with the same name as it’s corresponding component. (It’s a convention)
  • import that file in the component and use classes as in usual HTML
/* Heading.css */
.heading {
  color: red;
}
import "./Heading.css";

const Heading = () => {
  return <div className="heading">YOUR APP NAME</div>;
};

export default Heading;
CSS modules

Better way to write CSS for components.

  • used to prevent css classes from clashing and overwiting.
    /* App.css */
.heading {
  background-color: #888;
}
    import "./App.css";
import Heading from "./Heading";

function App() {
  return (
    <>
      <Heading />
    </>
  );
}

export default App;

    /* Heading.css */
.heading {
  color: red;
}
    import "./Heading.css";

const Heading = () => {
  return <div className="heading">YOUR APP NAME</div>;
};

export default Heading;

    Output:

    In CSS, the styles cascade, which means that styles can be inherited from parent elements.
    To avoid this kind of unintended styling, it's a good practice to use unique class names or use more specific selectors.

  • using css modules
    • rename the css file.

      Heading.css to Heading.module.css
    • change the import statement
      from
      import "./Heading.css";

      to import styles from "./Heading.module.css";
    • styles is a js object that has all the classes we have defined in the Heading.module.css file as it’s properties.
    • Add these styles’ properties to an element:

      <div className={styles.heading}>YOUR APP NAME</div>;
    /* Heading.module.css */
.heading {
  color: red;
}
    import styles from "./Heading.module.css";

const Heading = () => {
  return <div className={styles.heading}>YOUR APP NAME</div>;
};

export default Heading;

    Output:

    If you inpect it:
    <div class="
    _heading_1bb2y_1">YOUR APP NAME</div>
    and
    ._heading_1bb2y_1 {
    color: red;
    }

    class is not heading, it’s encoded.

    • It’s preferred to use camelCasing when writing css classes inside this css module file. Or else we will have to use square brackets (instead of dot notation) to access those classes/ styles’ properties. className={styles[main-heading]}
  • working of css modules
    • When building our application, build tools (like webpack, vite, nextJS) tranforms our class names and generates unique class names that don’t clash.
      So, we never have to worry about name clashes anymore.
    • CSS Modules enable local scoping of styles. Each React component gets its own unique CSS class names, ensuring that styles don't leak or clash with other components.
    • Since class names are scoped locally, you can use simple and descriptive names

Tailwind

CSS framework which uses utility classes for styling.
utility class- a CSS class written for one single purpose.

  • for installation, go to taiwind → installation → framework guides.
    also install
    Tailwind CSS intelliSence extension for vsCode
  • Responsiveness:
    By default, tailwind uses mobile first breakpoint system.
    what that means is
    • ‘sm’ breakpoint (i.e 640px) will translate to
      @media (min-width: 640px) {
  /* CSS styles go here */
}

      which means CSS will only be applied when the viewport(screen) width is 640 pixels or wider.

    • unprefixed utilities take effect on all screen sizes, while prefixed utilities take effect only take effect at the specified breakpoint or above
      • Example

        To center text in smaller screens

        • <div class="sm:text-center"></div> ❌
          this will center the text for screens equal to and larger than 640px. On smaller screens, it will be left aligned (default behavior).
        • <div class="text-center sm:text-left"></div> ✅
          this will center text for smaller mobile size screens and left align it on screens 640px or wider
    • sm: 640 px and wider : tablet
      lg: 1024 px and wider : laptop
      xl: 1280 px and wider : desktop

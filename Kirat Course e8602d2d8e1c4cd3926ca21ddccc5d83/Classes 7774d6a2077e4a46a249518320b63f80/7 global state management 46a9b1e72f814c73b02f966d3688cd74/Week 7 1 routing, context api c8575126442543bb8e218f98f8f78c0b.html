Routing

  • Jargon
    • Single page application
      • Before react, if a user wanted to switch to a different page in any application,
        say to see notifications page, upon clicking on the notifications button, browser would send another request to the back-end server and a new html, css and js files would come and a new page would be rendered.
        There would be a hard reload of the page.
      • After react, only the first time user gets a html, css and js files when he goes to any url. To change pages no new html, js, etc files are coming. No hard reloads are happening , thus called Single page application.
      • We can still do opimizations, so that the bundle for notifications page comes after user clicks on the notifications button (lazy loading of components).
    • Client side bundle
      • the js file that is recieved by the browser, which contains whole code for the app, is called Client side bundle
    • Client side routing
      • showing pages(components) based on what page(front-end routes) user is in
  • react-router-dom

Prop-drilling

It refers to the practice of passing down data from a higher-level component to a lower-level component through multiple layers of intermediate components.

Context Api

It doesn’t reduce re-renders but help eliiminate prop-drilling

  • Create a Context object using createContext() function
    import { createContext } from 'react';

export const CountContext = createContext();
  • Wrap Components with a Provider
    function App() {
  const [count, setCount] = useState(0);

// wrap any component that wants to use tha teleported values inside a provider
  return (
    <div>
      <CountContext.Provider value={{ count, setCount }}>
        <Counter />
      </CountContext.Provider>
    </div>
  );
}

// pass an object through value prop
  • Use the context in other components using useContext() hook
    // useContext hook returns what was passed inside the value prop of the provider
function Counter() {
  const { count } = useContext(CountContext);
  return (
    <>
      <h4>{count}</h4>
      <Buttons />
    </>
  );
}

    // button component uses setCount method, extracted from provider using useContext
function Buttons() {
  const { setCount } = useContext(CountContext);

  return (
    <>
      <button
        onClick={() => {
          setCount((prev) => prev + 1);
        }}
      >
        Increase
      </button>

      <button
        onClick={() => {
          setCount((prev) => prev - 1);
        }}
      >
        Decrease
      </button>
    </>
  );
}

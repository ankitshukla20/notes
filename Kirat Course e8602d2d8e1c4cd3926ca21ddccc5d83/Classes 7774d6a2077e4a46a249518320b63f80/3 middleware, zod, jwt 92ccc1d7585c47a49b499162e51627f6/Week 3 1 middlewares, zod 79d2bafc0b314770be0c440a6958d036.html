Middlewares

  • an express app is essentially a series of middleware function calls
    fn1(req, res, next) {
	next();
}
fn2(req, res, next) {
	next();
}
fn3(req, res, next) {
	next();
}

app.get("/", fn1, fn2, fn3, (req, res) => {})

    fn1, fn2, fn3 are middlewares and they are called in order they are passed inside any route handler

  • middlewares have access to request object(req), response object(res) and next middleware function in the application’s request-response cycle
  • middlewares can
    • terminate a request by sending a response or
    • pass control to next middleware by calling next() fn

app.use()

  • using app.use() we can apply a middleware to all the requests that are sent to the server
  • example
    // count number of requests sent to server

let requests = 0;
const numberOfRequests = () => {
	requests++;
	console.log(requests);
	next();
}

app.use(numberOfRequests);
  • app.use(express.json());
    • express.json() doesn’t return a value (0, 1 , etc), it returns a function itself, therefore we write it like express.json() and not express.json

Error handling middleware (global catch)

  • it needs to be definded at last, after all the route handlers.
  • code:
    // after all route handlers
app.use((err, req, res, next) => {
	let errorMessage = "An unknown error ooccured";
	let statusCode = 500;

	res.status(statusCode).json({error: errorMessage});
});
  • global catches help you give the user a better error message
  • wherever an exception is thrown, no more middlewares will be called after that and req will directly move to the global catch

Input validation (zod)

  • zod is an input validation library we use to avoid writing a list of checks for the inputs that are sent by the user.
  • code
    const {z} = require(”zod”);

app.get("/", (req, res) => {
	const email = req.body.email;
	
	// input validation
	const schema = z.string().email();	
	const inputValidation = schema.safeParse(email);

	if (!inputValidation.success) {
		res.status(400).json({message: "Your inputs are invalid"})
	}

	// ...code if inputs are valid
})
  • schemas examples
    • array of numbers
      const schema = z.array(z.number());
const inputValidation = schema.safeParse(arr);
    • object with email and password
      const schema = z.object({
	email: z.string().email(),
	password: z.string().min(8)
})

const inputValidation = schema.safeParse(obj);
  • parse and safeParse methods
    • parse - throws an exception if there is validation error
    • safeParse - returns an object that contains the result of validation

Authentication

  • Hashing
    • one-directional
    • with a given output, no one can find the input
  • Encryption
    • it is two-way
    • string is encripted using an enciption-key
    • string can be decripted using the same encription-key
  • Json web tokens
    • It is neither hashing not encription
    • With a given jwt any one can see the contents/ payload from which that token was created i.e. jwt can be decoded by anyone. (website: jwt.io)
    • 2 jwts can have same payload but they will be different if the secret used to create those 2 jwts were different
    • Understanding with example
      const payload = {
	username: "Ankit",
}

// server1
const secret1 = "secret1";
const token1 = jwt.sign(payload, secret1);
const payload = jwt.verify(token1, secret1);

// server2
const secret2 = "secret2";
const token2 = jwt.sign(payload, secret2);
const payload = jwt.verify(token2, secret2);
      • though token1 and token2 have same payload they will be different because of different secrets.
      • if any token with same payload but signed with any other secret than secret1 is sent to server1 it would not work. verify() method checks if the token given was signed with the given secret.
      • if anyone doesn’t know the secretKey of the server, even if they have the idea of payload they can’t send a correct token without the knowledge of secretKey. If they send a token signed by any other secret, verify() method will throw an exception.
    • 3 parts of jwt
      • Header: Contains the type of token and the signing algorithm.

        Example: {"alg": "HS256", "typ": "JWT"}

      • Payload: Contains claims (statements about an entity).

        Example: {"username": "Ankit", "iat": 1516239022}

      • Signature: Generated by combining encoded header, encoded payload, and a secret key using the specified algorithm.

        Example: HMACSHA256(base64UrlEncode(header) + "." + base64UrlEncode(payload), secret)

      • when combined, the 3 parts form a complete jwt

  • Use mongoose (a library) to connect backend server with mongoDB (a noSql DB)
    mongoose

  .then(() => {
    console.log("Connection to the database is established");

  })
  .catch((err) => console.log("Error connecting to the database"));
  • Using Mongoose
    • create a schema using mongoose.schema().
      Schema is a blueprint/ structure of the data which we will be storing in mongoDb
      const todoSchema = mongoose.Schema({
  title: String,
  description: String,
  completed: Boolean,
});

    • create a model using mongoose.model()
      A model is a class with which we construct documents
      const Todo = mongoose.model("Todo", todoSchema);

      const Todo = mongoose.model("Todo", todoSchema);

      • Todo - class we will use to construct documents
      • Todo - name of the collection in the singular form, mongoose will automatically create collection in the DB with lower case plural form name. Here collection will be created with todos name
      • constructed documents are stored in this collection

    • Implement CRUD operations using Model APIs, refer mongoose documentation.
